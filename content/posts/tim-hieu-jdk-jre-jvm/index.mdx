Sau khi đọc xong, bạn sẽ biết cách phân biệt, lựa chọn và sử dụng đúng các thành phần này trong quá trình lập trình Java.

## Nội dung chính

### JDK, JRE, JVM là gì? (Tìm hiểu về JDK, JRE, JVM)

- **JDK (Java Development Kit):** Bộ công cụ dành cho lập trình viên Java, bao gồm trình biên dịch (javac), thư viện chuẩn, công cụ debug, và cả JRE.
- **JRE (Java Runtime Environment):** Môi trường chạy ứng dụng Java, gồm JVM và các thư viện cần thiết để chạy chương trình đã biên dịch.
- **JVM (Java Virtual Machine):** Máy ảo Java, chịu trách nhiệm thực thi bytecode Java trên nhiều nền tảng khác nhau.

#### Minh họa bằng sơ đồ

```
+-------------------+
|    JDK            |
|  +-------------+  |
|  |   JRE       |  |
|  | +--------+  |  |
|  | | JVM    |  |  |
|  | +--------+  |  |
|  +-------------+  |
+-------------------+
```

### Quá trình chạy một chương trình Java

Khi bạn viết mã Java, quá trình chạy sẽ diễn ra như sau:

1. **Viết mã nguồn (.java)**
2. **Biên dịch bằng JDK (javac) → Tạo file .class (bytecode)**
3. **JVM trong JRE thực thi bytecode**

```java
// Ví dụ đơn giản về chương trình Java
public class HelloWorld {
  public static void main(String[] args) {
    System.out.println("Xin chào, Java và JVM!");
  }
}
```

### Khi nào cần JDK, khi nào chỉ cần JRE?

- **Lập trình viên:** Cần cài đặt JDK để có thể biên dịch và chạy chương trình Java.
- **Người dùng cuối:** Chỉ cần JRE để chạy các ứng dụng Java đã được biên dịch sẵn.

#### So sánh nhanh

| Thành phần | Dùng để phát triển | Dùng để chạy ứng dụng |
| ---------- | ------------------ | --------------------- |
| JDK        | ✔️                 | ✔️                    |
| JRE        | ❌                 | ✔️                    |
| JVM        | ❌                 | ✔️                    |

### Tìm hiểu về JDK, JRE, JVM qua ví dụ thực tế

Giả sử bạn tải một ứng dụng Java về máy, nếu chỉ muốn chạy ứng dụng, bạn chỉ cần JRE. Nhưng nếu muốn sửa mã nguồn, biên dịch lại, bạn cần JDK.

```bash
# Kiểm tra phiên bản JDK
javac -version

# Kiểm tra phiên bản JRE/JVM
java -version
```
